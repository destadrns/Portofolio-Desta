# PRD — Personal Portfolio Website

## 1. Product Overview

This product is a personal portfolio website designed to introduce the website owner, showcase skills, present selected projects, explain the learning journey, and provide professional contact information.

The website should feel calm, personal, interactive, modern, and visually polished without looking overly generic or obviously AI-generated. The design must avoid stiff typography, rigid cards, and overly corporate layouts. It should feel like a custom-made portfolio for an Indonesian Informatics student who is seriously growing as a web developer.

Important language requirement:

* The PRD is written in English for better AI agent execution.
* All public-facing website content must be written in Bahasa Indonesia.
* Navigation labels, hero copy, section titles, project descriptions, button text, and contact copy should use natural Indonesian.
* Avoid robotic, overly formal, or template-like Indonesian sentences.

## 2. Product Goals

The website should achieve the following goals:

1. Present a personal identity in a professional but natural way.
2. Showcase web development skills, learning progress, and selected projects.
3. Build a calm and trustworthy personal brand.
4. Make the website feel interactive without using excessive animation.
5. Avoid an AI-generated or template-like impression.
6. Provide a clean structure that can be expanded later with more projects, certificates, and experiences.

## 3. Target Users

The primary users are:

* Recruiters or HR staff.
* Lecturers or academic reviewers.
* Friends, collaborators, and developer communities.
* Potential small clients.
* The portfolio owner for long-term personal branding.

## 4. Brand Personality

The website should communicate these qualities:

* Calm
* Clean
* Honest
* Friendly
* Focused
* Modern
* Personal
* Still growing
* Not overclaiming
* Professional but not stiff

The portfolio should not present the owner as an exaggerated “expert senior developer.” Instead, it should present the owner as someone who is learning seriously, building real projects, and improving through practice.

## 5. Design Direction

### 5.1 Main Design Concept

Use the concept:

**Calm Interactive Personal Portfolio**

The design should feel soft, warm, modern, and comfortable to explore. It should not feel like a typical AI-generated portfolio template with generic gradients, stiff cards, and exaggerated text.

The design should use:

* Soft calm colors.
* Rounded and friendly cards.
* Natural spacing.
* Smooth micro-interactions.
* Personal Indonesian copywriting.
* Clean section hierarchy.
* Subtle decorative elements.
* Lightweight animations.

Avoid:

* Harsh neon colors.
* Overly futuristic design.
* Generic developer illustrations.
* Too many glowing effects.
* Excessive glassmorphism.
* Overly stiff rectangular cards.
* Generic AI-sounding text.

The typography must feel modern, soft, and not too rigid.

Recommended fonts:

## 8. Content Language Guidelines

All visible website content must be in Bahasa Indonesia.

The Indonesian copy should sound natural, personal, and not too formal. Avoid generic AI phrases.

Avoid sentences like:

* “Saya adalah pribadi yang kreatif, inovatif, dan komunikatif.”
* “Saya menciptakan solusi digital terbaik untuk masa depan.”
* “Saya adalah developer yang passionate dalam membuat pengalaman digital luar biasa.”
* “Saya siap memberikan kontribusi maksimal dengan kemampuan terbaik saya.”

Use more natural sentences like:

* “Saya sedang membangun kemampuan di bidang web development dengan fokus pada tampilan yang rapi, responsif, dan nyaman digunakan.”
* “Saya suka membuat project yang tidak hanya berjalan, tetapi juga mudah dipahami dan enak digunakan.”
* “Saat mengerjakan project, saya biasanya memecah masalah menjadi bagian kecil agar prosesnya lebih terarah.”
* “Saat ini saya terus memperkuat pemahaman dalam React, struktur komponen, dan desain antarmuka.”

The tone should be:

* Honest
* Specific
* Warm
* Not exaggerated
* Not too stiff
* Suitable for a student portfolio

## 10. UI Component Guidelines

### 10.1 Cards

Cards must feel soft and modern.

Card requirements:

* Border radius: 20px–28px.
* Soft border.
* Light shadow.
* Comfortable padding.
* Subtle hover state.
* No stiff sharp rectangles.
* No heavy shadow.
* No excessive glassmorphism.

Cards should feel friendly and custom-made.

### 10.2 Buttons

Button types:

1. Primary button
2. Secondary button
3. Text button

Button style:

* Rounded shape.
* Calm accent color.
* Smooth transition around 200ms–300ms.
* Clear hover state.
* Clear focus state.
* No overly aggressive colors.

### 10.3 Icons

Recommended icon libraries:

* Lucide React
* Phosphor Icons
* Tabler Icons

Icon style:

* Outline icons.
* Consistent size.
* Soft color.
* Not too many icons in one section.

### 10.4 Animation

Animations should be subtle and smooth.

Allowed animations:

* Fade in.
* Slide up.
* Hover lift.
* Smooth scroll.
* Soft background blob movement.
* Navbar blur on scroll.
* Button hover micro-interaction.

Avoid:

* Fast animations.
* Too many moving objects.
* Heavy 3D animations.
* Strong neon glow.
* Annoying cursor effects.
* Long loading animation.

## 11. Responsiveness

The website must be fully responsive.

Breakpoints:

* Mobile: 360px and above.
* Tablet: 768px and above.
* Desktop: 1024px and above.
* Large screen: 1440px and above.

Mobile behavior:

* Navbar becomes hamburger menu.
* Hero section becomes single column.
* Cards stack vertically.
* Font size adjusts properly.
* Buttons remain easy to tap.
* No horizontal scrolling.
* Section spacing remains comfortable.

## 12. Accessibility Requirements

The website should follow basic accessibility standards.

Requirements:

* Good text contrast.
* Buttons must have focus states.
* Links must be keyboard accessible.
* Images must have alt text.
* Heading hierarchy must be correct.
* Do not rely only on color to communicate information.
* Animations should not be distracting.
* Use semantic HTML when possible.

## 13. Performance Requirements

The website must feel lightweight and fast.

Targets:

* Lighthouse Performance score: minimum 85.
* Optimized images.
* Minimal unnecessary dependencies.
* Lazy load project images.
* Use only necessary animations.
* Avoid heavy libraries unless needed.
* Keep CSS and components clean.

## 15. Recommended Tech Stack

Recommended stack:

* React
* Vite

Alternative simple stack:
